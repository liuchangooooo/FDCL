# 真正有挑战的障碍物场景设计

## 核心思想

**简单绕行 ≠ 有挑战**

真正的挑战应该来自：
1. **姿态约束**：某些朝向无法通过
2. **空间约束**：必须在狭小空间内调整
3. **顺序约束**：必须先做 A 才能做 B
4. **精确约束**：容错空间极小

---

## 场景类型重新分类

### 类型 1：强制旋转场景

**核心**：T-block 当前朝向无法到达目标，必须先旋转

#### 场景 1.1：窄门旋转
```
初始状态：T-block 横杠朝右 (0°)
目标：到达 (0, 0)，朝向 -45°

障碍物布局：
┌─────────────────────────────────────┐
│                                     │
│    🔵 T-block (横杠朝右，10cm)      │
│    ├─┤                              │
│     │                               │
│     │                               │
│     │      ●           ●            │
│     │       ╲         ╱             │
│     │        ╲  9cm  ╱  ← 窄门      │
│     │         ╲     ╱               │
│     ▼          ╲   ╱                │
│              🟢 Target              │
└─────────────────────────────────────┘

关键约束：
- 门宽 9cm < 横杠 10cm
- 必须先旋转到竖杠朝前 (7cm < 9cm)
- 通过后再旋转到目标朝向 -45°

技能测试：
- 旋转控制
- 空间推理（判断哪个朝向能通过）
- 多步规划
```

│     │           ●                   │

### 类型 2：精确操作场景

**核心**：容错空间极小，必须精确控制

#### 场景 2.1：之字形通道
```
┌─────────────────────────────────────┐
│    🔵                               │
│     │                               │
│     │  ●●●●●●●                      │
│     │         ●                     │
│     │         ●  ← 11cm 宽          │
│     ▼         ●●●●●●●               │
│                      ●              │
│                      ●  ← 11cm 宽   │
│              ●●●●●●●●               │
│                    🟢               │
└─────────────────────────────────────┘

关键约束：
- 通道宽度 11cm，T-block 宽度 10cm
- 容错空间只有 1cm
- 任何偏差都会碰撞
- 需要多次精确转向

技能测试：
- 精确路径跟踪
- 连续精确操作
- 碰撞避免
```

### 类型 3：顺序约束场景

**核心**：必须按特定顺序操作，否则会陷入死局

#### 场景 3.1：推箱子式
```
┌─────────────────────────────────────┐
│    🔵                               │
│     │                               │
│     │    ●                          │
│     │   ╱ ╲                         │
│     │  ╱   ╲                        │
│     │ ╱  A  ╲  ← 区域 A（狭窄）     │
│     ▼╱       ╲                      │
│     ╱         ╲                     │
│    ╱           ●                    │
│   ╱           ╱ ╲                   │
│  ●           ╱ B ╲  ← 区域 B        │
│             ╱     ╲                 │
│            ╱   🟢  ╲                │
└─────────────────────────────────────┘

关键约束：
- 如果先进入区域 B，会被困住
- 必须先从区域 A 绕行
- 需要提前规划整体路径

技能测试：
- 全局路径规划
- 死局识别
- 长期规划能力
```

## 难度分级（基于真实挑战）

### Level 1-2：基础（仍然可以简单绕行）
```
- 外围障碍物
- 宽通道（>15cm）
- 单一路径阻挡
```

### Level 3-4：需要旋转
```
- 窄门旋转（9-11cm）
- L 型通道
- 简单的姿态调整
```

### Level 5-6：需要精确控制
```
- 之字形通道
- 螺旋接近
- 容错空间 <2cm
```

### Level 7：需要复杂规划
```
- 推箱子式顺序约束
- 单向门
- 多阶段组合约束
```

## 关键设计原则

### 1. 利用 T 形的不对称性
```
横杠 10cm vs 竖杠 7cm
→ 设计 8-9cm 的通道
→ 强制旋转
```

### 2. 利用推动的方向性
```
只能推，不能拉
→ 设计单向通道
→ 强制顺序
```

### 3. 利用空间的有限性
```
桌面 50cm × 50cm
→ 设计复杂路径
→ 强制精确控制
```

### 4. 利用目标的朝向要求
```
目标朝向 -45°
→ 设计需要多次旋转的场景
→ 测试姿态控制
```

## LLM Prompt 改进

### 错误的 Prompt（导致简单绕行）
```
"在路径中点放置障碍物，阻挡直线推动"
→ LLM 只会放几个点
→ 绕一下就过去了
```

### 正确的 Prompt（创造真实挑战）
```
"设计一个场景，使得：
1. T-block 当前朝向（横杠朝右）无法直接到达目标
2. 必须先旋转到竖杠朝前，才能通过某个关键通道
3. 通道宽度 9cm（小于横杠 10cm，大于竖杠 7cm）
4. 通过后还需要再次旋转到目标朝向 -45°

请给出障碍物的具体坐标，形成这样的约束。"
```

## 总结

**简单绕行的场景没有意义**

真正有挑战的场景应该：
1. **强制旋转**：利用 T 形不对称性
2. **强制精确**：容错空间极小
3. **强制顺序**：错误的顺序会陷入死局
4. **强制组合**：多种技能同时需要

这样才能真正测试策略的能力，而不是简单的"绕一下"。
