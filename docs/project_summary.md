## ⚠️ 常见陷阱与避免方法

### 陷阱1: 数据泄露
**问题**: 训练集和测试集来自同一文件，导致过高准确率

**解决**:
- 参考arXiv 2024论文正确划分数据
- 确保测试集完全独立
- 可以按文件划分，而不是随机划分

### 陷阱2: 过度拟合
**问题**: 训练准确率99%，测试准确率70%

**解决**:
- 使用Dropout（0.5）
- 数据增强（噪声注入、时间平移）
- 早停机制
- 正则化（L2）

### 陷阱3: 模型太复杂
**问题**: 堆砌太多技术，无法解释清楚

**解决**:
- 优先保证基础功能
- 每个模块都要能解释
- 循序渐进，先简单后复杂

### 陷阱4: 缺乏对比
**问题**: 只有一个模型，无法证明有效性

**解决**:
- 至少实现2-3个模型
- 包含传统方法作为baseline
- 充分的消融实验

## 🎓 答辩准备

### 可能的问题及回答

#### Q1: 为什么选择这个数据集？
**A**: CWRU数据集是故障诊断领域的标准数据集，被广泛用于验证新算法。它包含多种故障类型和工况，具有代表性。同时数据公开可获取，便于复现和对比。

#### Q2: 你的创新点是什么？
**A**: （根据选择的方案回答）
- 方案A: 设计了多尺度卷积+注意力机制，能够自适应提取不同粒度的故障特征
- 方案B: 提出CNN-Transformer混合架构，结合局部特征和全局依赖
- 方案C: 首次将Vision Transformer应用于振动信号故障诊断，并验证了跨工况泛化能力

#### Q3: 准确率为什么这么高？
**A**: 
1. CWRU数据集信噪比较高，故障特征明显
2. 深度学习能够自动学习判别性特征
3. 我们使用了数据增强和正则化防止过拟合
4. 通过充分的调参优化了模型性能

#### Q4: 能否实际应用？
**A**: 
1. 模型推理时间<100ms，满足实时性要求
2. 我们设计了轻量化版本，可部署到边缘设备
3. 通过迁移学习验证了跨工况泛化能力
4. 未来可以结合工业物联网实现在线监测

#### Q5: 有什么不足和改进方向？
**A**:
1. 目前只在CWRU数据集上验证，未来可在更多数据集上测试
2. 可以加入不确定性估计，给出置信度
3. 可以研究小样本学习，减少对标注数据的依赖
4. 可以结合物理模型，提高可解释性
