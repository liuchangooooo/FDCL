# 高级难度系统设计

## 概述

从 3 级扩展到 **7 级难度系统**，每一级都有明确的技能目标和位置策略。

---

## 1. 难度级别总览

| Level | 名称 | 障碍物数 | 核心策略 | 目标技能 | 通道宽度 |
|-------|------|---------|---------|---------|---------|
| 1 | 入门 | 1 | 外围干扰 | 基本避障意识 | >18cm |
| 2 | 简单 | 1-2 | 偶尔阻挡 | 基本路径规划 | 15cm |
| 3 | 中等偏易 | 1-3 | 经常阻挡 | 路径规划+精确控制 | 13cm |
| 4 | 中等 | 2-3 | 多重约束 | +旋转控制 | 12cm |
| 5 | 中等偏难 | 2-4 | 复杂约束 | +多步推理 | 11cm |
| 6 | 困难 | 3-5 | 高级约束+陷阱 | +错误恢复 | 10.5cm |
| 7 | 专家 | 3-6 | 动态约束+陷阱 | 完美执行 | 10.2cm |

**动态约束说明**:
- 根据 T-block 的初始位置和朝向，LLM 自主设计最具挑战性的配置
- 例如：如果 T-block 距离目标很近，则在目标周围密集放置障碍物
- 如果 T-block 距离目标很远，则在路径中段设置多重关卡

## 3. 位置策略详解

### 3.1 外围干扰 (Peripheral)
```
特点: 障碍物在角落，不影响路径
适用级别: Level 1-2
```

### 3.2 路径阻挡 (Path Blocking)
```
特点: 阻挡直线路径，但有明显绕行
适用级别: Level 2-4
```

### 3.3 狭窄通道 (Corridor)
```
特点: 两个障碍物形成通道，需要精确通过
适用级别: Level 3-6
通道宽度: 15cm → 13cm → 12cm → 11cm → 10.5cm
```

### 3.4 包围目标 (Surrounding)
```
特点: 障碍物围绕目标，只留一个进入角度
适用级别: Level 4-7
```

### 3.5 旋转挑战 (Rotation Required)
```
特点: 当前朝向无法通过，必须先旋转
适用级别: Level 4-7
```

### 3.6 多步规划 (Multi-Step)
```
特点: 无法直接推到目标，需要中间调整
适用级别: Level 5-7
```

### 3.7 陷阱 (Trap)
```
特点: 有看似可行但实际是死路的路径
适用级别: Level 6-7
目的: 测试错误恢复能力
```

### 3.8 动态约束 (Dynamic)
```
特点: LLM 根据 T-block 位置自适应设计
适用级别: Level 7
目的: 极限挑战
```

## 4. 升级决策逻辑

### 4.1 多指标综合评估

```python
升级条件 = (
    成功率 ≥ 阈值 AND
    碰撞率 ≤ 阈值 AND
    平均步数 ≤ 阈值 AND
    训练 episodes ≥ 最小值
)

降级条件 = (
    成功率 < 阈值 OR
    碰撞率 > 阈值
)
```

### 4.2 渐进式升级

```
满足升级条件 → upgrade_counter++
counter ≥ 2 → 真正升级
```

### 4.3 难度变化曲线

## 6. 使用示例

```python
from DIVO.env.pusht.advanced_difficulty_system import (
    AdvancedDifficultyManager,
    PlacementStrategy
)

# 初始化
manager = AdvancedDifficultyManager(initial_level=1)

for episode in range(1000):
    # 获取当前配置
    config = manager.get_current_config()
    strategy = manager.get_placement_strategy()
    
    print(f"Level {config.level}: {config.name}")
    print(f"策略: {strategy.value}")
    print(f"障碍物数量: {config.num_obstacles_range}")
    print(f"目标技能: {[s.value for s in config.target_skills]}")
    
    # 执行 episode
    success, steps, collision = run_episode(...)
    
    # 记录结果
    manager.record_episode(success, steps, collision)
    
    # 定期评估
    if (episode + 1) % 50 == 0:
        new_level, eval_result = manager.evaluate_and_update()
        print(f"评估结果: {eval_result['action']}")
```

## 7. 总结

### 7.1 核心优势

1. **细粒度控制**: 7 个级别，每级都有明确的目标
2. **多维度评估**: 不只看成功率，综合评估多个指标
3. **渐进式升级**: 避免难度跳跃，确保稳定学习
4. **丰富的策略**: 8 种位置策略，覆盖各种场景
5. **自适应调整**: 根据表现动态调整难度

### 7.2 技能树

```
Level 1: 基本避障意识
  ↓
Level 2: + 基本路径规划
  ↓
Level 3: + 精确控制
  ↓
Level 4: + 旋转控制 + 空间推理
  ↓
Level 5: + 多步推理
  ↓
Level 6: + 错误恢复 + 效率优化
  ↓
Level 7: 完美执行 (所有技能的综合)
```

### 7.3 预期训练时长

```
Level 1: 30-50 episodes
Level 2: 50-80 episodes
Level 3: 80-120 episodes
Level 4: 100-150 episodes
Level 5: 120-180 episodes
Level 6: 150-220 episodes
Level 7: 200+ episodes

总计: 约 730-1000 episodes 达到专家级
```

这个系统提供了一个完整的、可扩展的难度管理框架，能够支持从入门到专家的完整训练过程。
