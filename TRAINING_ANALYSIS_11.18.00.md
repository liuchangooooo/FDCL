## 8. 核心问题诊断

### 问题现象
1. ⚠️ **成功率极低** (0.00%)，几乎所有 episode 都失败
2. ⚠️ **碰撞率极高** (79.88%)，策略频繁撞到障碍物
3. ⚠️ **奖励停滞**在 -10 左右，没有明显提升
4. ⚠️ **Episode 极短** (平均 1.07 steps)，说明很快就碰撞失败

### 可能原因

#### 1. LLM 生成的障碍物过于困难
- Phase 0 应该是"简单"难度，但实际碰撞率 79.88%
- 可能 LLM 生成的障碍物布局不合理：
  - 太密集
  - 太靠近起点
  - 阻挡了直线路径

#### 2. 拓扑生成器代码质量问题
- Stage A 生成的代码可能没有遵循 Phase 0 的"简单"原则
- 可能没有正确使用 is_safe() 检查碰撞
- 可能障碍物间距不足或位置不合理

#### 3. 训练时间不足
- 虽然训练了 145k epochs，但在困难环境中可能需要更长时间
- 成功率 0.0015% 说明策略几乎没有学到有效的避障策略

#### 4. 奖励函数问题
- 奖励在 -10 左右，可能是碰撞惩罚过大
- 策略可能陷入局部最优（不敢移动以避免碰撞）

```bash
  curriculum.enable_acgs_loop=True
```
- 让 LLM 根据失败数据自动调整障碍物难度
- 从当前的 79.88% 碰撞率逐步降低到合理水平

1. **改进 prompt 设计**
   - 使用更强的引导（如 Stage A compact prompt）

   - 在生成后验证障碍物难度

## 11. 总结

这次训练暴露了一个关键问题：**LLM 生成的障碍物难度远超 Phase 0 的预期**。

虽然训练过程本身是稳定的（网络收敛、Q 值准确、潜在空间分布良好），但环境过于困难导致策略无法学习。79.88% 的碰撞率和 1.07 steps 的平均 episode 长度说明障碍物几乎立即阻挡了推动。

**关键发现**：
- ACGS 数据收集系统工作正常
- 训练基础设施稳定可靠
- 但需要改进 LLM prompt 或启用 ACGS 闭环来自动调整难度
